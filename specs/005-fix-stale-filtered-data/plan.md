# Implementation Plan: Fix Stale Filtered Data on Cold Start

**Branch**: `005-fix-stale-filtered-data` | **Date**: 2026-03-05 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-fix-stale-filtered-data/spec.md`

## Summary

On cold start, `LaunchRepositoryImpl` returns fresh cached launches **without applying user filter preferences** (agency/location). Filters are only applied on the stale cache path and API path, but the fresh cache path (`getUpcomingNormalLaunches`) returns raw SQL results directly. Additionally, both Android and iOS widgets never pass filter preferences to their repository calls. The fix applies `filterLaunchesByPreferences()` to the fresh cache path and integrates `LaunchFilterService` into widget data fetching.

**Constraints**: Must not change `LaunchRepository` interface signatures; offline-capable cache behavior preserved
**Scale/Scope**: 5 files modified (common + Android + iOS), ~50 lines changed
