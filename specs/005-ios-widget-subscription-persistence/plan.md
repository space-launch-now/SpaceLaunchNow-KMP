# Implementation Plan: iOS Widget Subscription Persistence

**Branch**: `005-ios-widget-subscription-persistence` | **Date**: 2026-03-05 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-ios-widget-subscription-persistence/spec.md`

## Summary

Paid iOS users experience locked widgets after force-closing the app because the current implementation only caches a simple boolean (`widget_has_access`) that requires the main app to be running to update. This plan implements a robust caching system that stores subscription expiry dates and premium history, enabling widgets to make fail-safe access decisions without the main app running.

**Technical Approach**: Enhance the App Group `NSUserDefaults` cache with expiry timestamps and a "was ever premium" flag. Implement fail-safe logic in the widget that defaults to unlocked for users who have ever been premium, only locking when subscription definitively expires.

## Phase 0 Research Summary

See [research.md](research.md) for full details.

**Key Findings**:
1. Current cache only stores boolean - insufficient for offline validation
2. RevenueCat provides expiry dates via `EntitlementInfo.expirationDateMillis`
3. Widget extensions should avoid initializing heavy frameworks
4. App Group NSUserDefaults is the right storage mechanism

**Decisions Made**:
- Add 4 new keys to App Group NSUserDefaults (expiry, lastVerified, wasEverPremium, subscriptionType)
- Implement sticky `wasEverPremium` flag that never reverts to false
- Default to unlocked for ambiguous states where user has premium history

## Phase 1 Design Summary

See [data-model.md](data-model.md) for full details.

**New Data Structures**:
1. `WidgetAccessCache` (Kotlin) - Cache data transfer object
2. `WidgetAccessState` (Swift) - Cache reader with fail-safe logic
3. Enhanced `LocalSubscriptionData` - Add `subscriptionExpiryMs: Long?` field

**Fail-Safe Logic**:
```
IF hasAccess == true → UNLOCKED
ELSE IF wasEverPremium == false → LOCKED
ELSE IF subscriptionExpiry > now → UNLOCKED
ELSE IF subscriptionExpiry < now → LOCKED
ELSE IF wasEverPremium == true → UNLOCKED (grace period)
```

## Verification Checklist

- [ ] Premium user force closes app → Widget stays unlocked
